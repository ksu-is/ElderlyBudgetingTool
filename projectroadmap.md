# Project Roadmap

## Project Title
Simple Budgeting Software for Elderly Users

## Sprint 1 Work

### Code Base Reviewed
I explored the “AgeLink” Python project from the KSU-IS GitHub organization. This project is a web-based application designed to help elderly users connect with community resources.

### What I Did
I opened and reviewed several important files in the repository, including:
- app.py (main application logic)
- models.py (data handling)
- forms.py (user input forms)
- templates folder (HTML pages for the interface)

I read through the code to understand how the application works and how different parts connect together.

I also attempted to understand how the program would run as a web application using Python.

### What I Learned
From reviewing this code base, I learned:
- how Python applications can be structured into multiple files
- how user input is handled using forms
- how data is stored and processed
- how web applications use templates to display information
- how larger applications are organized compared to simple scripts

### Evaluation
This code base is more advanced than what I need for my project. It is designed as a full web application, which may be too complex for a simple budgeting tool for elderly users.

However, it is useful for understanding how real-world applications are built and organized.

### How This Helps My Project
This project helped me understand how to structure a larger Python application. For my budgeting app, I will:
- keep the design much simpler
- focus on easy input and output
- avoid unnecessary complexity
- prioritize usability for elderly users

### Attempt to Run
I reviewed the structure of the application and attempted to understand how it would run using Python. This helped me learn how different components of a Python project work together.

### Next Steps
- design a simple budgeting interface
- build income and expense tracking
- add automatic balance calculations
- keep the program easy to use and readable
